         │                  │                    │
         └──────────────────┼────────────────────┘
                            ▼
                    ┌─────────────────┐

                    │    (Extract)    │
                    └─────────────────┘
                            │
                            ▼

├───────────────────────────────────────────────────────────────────────────┤

STEP 1: SCRAPING
════════════════
orchestrate_full_pipeline.py::step_1_scrape_articles()
  │
  ├─ Fetch from 3 RSS feeds
  ├─ Parse XML → extract article metadata
  ├─ Check MongoDB for duplicates (by guid)
  ├─ Insert new articles
  │
  ▼
✓ Output: articles collection (MongoDB)
  Total docs: ~100-200 per run
  Status: Ready for WF1


STEP 2: CONTENT INTELLIGENCE (WF1)
═══════════════════════════════════
orchestrate_full_pipeline.py::step_2_create_insights()
  │
  ├─ Load wf1.py ContentIntelligencePipeline
  ├─ Read pending articles (status != analyzed)
  ├─ For each article:
  │  ├─ Call MegaLLM API with DeepSeek V3.1
  │  ├─ Extract marketing angles:
  │  │  ├─ hook_sentence (compelling opening)
  │  │  ├─ core_claim (main takeaway)
  │  │  ├─ megallm_tie_in (relevance to product)
  │  │  ├─ infra_data_point (technical detail)
  │  │  └─ angle_type (category classification)
  │  │
  │  └─ Insert into content_insights
  │
  ▼
┌───────────────────────────────────────────────────────────────────────────┐
│ MongoDB: content_insights (WF1 OUTPUT)                                   │
├───────────────────────────────────────────────────────────────────────────┤
│  {                                                                         │
│    "_id": ObjectId,                                                       │
│    "article_url": "https://....",                                        │
│    "hook_sentence": "Latest AI model achieves 90% speedup",             │
│    "core_claim": "New optimization technique saves 40% compute",        │
│    "megallm_tie_in": "Relevant to cost optimization",                   │
│    "infra_data_point": "Token efficiency, inference cost",              │
│    "angle_type": "performance",  /* or: outage, pricing, etc */         │
│    "value_prop": "CTO reduces infrastructure costs",                     │
│    "status": "pending_generation",  /* ready for blog generation */     │
│    "icp_score": 8.5,                                                     │
│    "created_at": "2024-01-15T10:30:00Z"                                │
│  }                                                                        │
│                                                                           │
│  Status: Awaiting blog generation                                        │
│  Count:  10-20 insights per run (depends on articles)                   │
└───────────────────────────────────────────────────────────────────────────┘


STEP 3: BLOG GENERATION (5 ACCOUNT DISTRIBUTION)
═════════════════════════════════════════════════
orchestrate_full_pipeline.py::step_3_generate_blogs_for_accounts()
       └─> InsightDrivenBlogScheduler (blog_platform/insight_scheduler.py)
           │
           ├─ Connect to MongoDB
           ├─ Fetch pending insights (status='pending_generation')
           │
           ├─ For each insight:
           │  │
           │  ├─ For account_1:
           │  │  ├─ Extract: hook_sentence, core_claim, infra_data_point
           │  │  ├─ Call OpenRouter API (FREE Qwen model)
           │  │  │  ├─ Model: qwen/qwen3.6-plus-preview:free
           │  │  │  ├─ API: https://openrouter.ai/api/v1
           │  │  │  └─ Response: {title, body}
           │  │  ├─ Add metadata: account_id, topic, status, created_at
           │  │  ├─ Insert to blogs collection
           │  │  └─ Return: blog_id
           │  │
           │  ├─ For account_2: [same process]
           │  ├─ For account_3: [same process]
           │  ├─ For account_4: [same process]
           │  └─ For account_5: [same process]
           │
           ├─ Mark insight status: 'pending_generation' → 'blogs_generated'
           │
           └─ Log summary

Example: 15 insights → 15 × 5 = 75 blogs generated

┌───────────────────────────────────────────────────────────────────────────┐
│ MongoDB: blogs (FINAL OUTPUT)                                             │
├───────────────────────────────────────────────────────────────────────────┤
│  {                                                                         │
│    "_id": ObjectId("..."),                                               │
│    "title": "How We Cut Inference Costs by 40%",                         │
│    "body": "Long-form blog content (600-900 words)...",                 │
│    "account_id": "account_1",  /* Distributed to 1 of 5 accounts */     │
│    "topic": "Cost Optimization",                                         │
│    "insight_id": "ObjectId(...)",  /* Links back to source insight */   │
│    "status": "draft",                                                    │

│  - One doc per (insight × account)                                      │

│                                                                           │
│  Example distribution:                                                   │
│  ├─ account_1: 15 blogs (one per insight)                              │
│  ├─ account_2: 15 blogs (one per insight)                              │
│  ├─ account_3: 15 blogs (one per insight)                              │
│  ├─ account_4: 15 blogs (one per insight)                              │
│  └─ account_5: 15 blogs (one per insight)                              │
│  ═════════════════════════════════════════                             │
│     Total: 75 blogs from 15 insights                                     │
└───────────────────────────────────────────────────────────────────────────┘

CORE COMPONENTS
═══════════════

1. orchestrate_full_pipeline.py (MAIN ORCHESTRATION)
   ├─ step_1_scrape_articles()         ◄─ Calls scrape_to_mongo
   ├─ step_2_create_insights()         ◄─ Calls wf1
   ├─ step_3_generate_blogs_for_accounts()  ◄─ Calls insight_scheduler
   ├─ print_summary()                  ◄─ Logs complete results
   └─ Runs sequentially with error handling

2. blog_platform/insight_scheduler.py (NEW - BLOG GENERATION ENGINE)
   ├─ class InsightDrivenBlogScheduler
   │  ├─ connect_mongo()                ◄─ Connect to MongoDB
   │  ├─ get_pending_insights()         ◄─ Fetch insights from DB
   │  ├─ generate_blogs_from_insight()  ◄─ Single insight → 5 accounts
   │  └─ generate_blogs_for_all_accounts() ◄─ All insights → All accounts
   └─ generate_blogs_from_insights_now() ◄─ Quick function for API

3. scrape_to_mongo.py (EXISTING - UNCHANGED)
   ├─ scrape_new_articles()            ◄─ Scrapes RSS feeds
   └─ Returns: {inserted, skipped}

4. wf1.py (EXISTING - UNCHANGED)
   ├─ ContentIntelligencePipeline
   │  └─ run()                          ◄─ Analyzes articles
   └─ Returns: {insights_created, errors}

5. blog_platform/blog_generator.py (EXISTING - UNCHANGED)
   ├─ class BlogGenerator
   │  ├─ generate_blog()                ◄─ Single blog via API
   │  └─ batch_generate()               ◄─ Multiple blogs
   └─ Uses: OpenRouter API
