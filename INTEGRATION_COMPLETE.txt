HOW IT WORKS
════════════

BEFORE (OLD SYSTEM):
  [Random Topics] → [OpenRouter API] → [5 Accounts] → [blogs collection]
  
  - Generated 12 blogs per account per day
  - All with random topics
  - Disconnected from WF1 analysis
  - No insight-based content

AFTER (NEW SYSTEM):
  [RSS Feeds] → [Scraper] → [articles]
                               ↓
                         [WF1 Analysis] → [content_insights]
                                            ↓
                        [5 Account Distribution] ← [Insight-driven]
                                ↓
                          [blogs collection]
  
  - Generates blogs from real WF1 insights
  - Each insight creates 5 blog variants
  - Fully integrated pipeline
  - Content based on actual analysis

CONFIGURATION
══════════════

Your .env should have:

  MONGODB_URI=mongodb://localhost:27017
  MONGODB_DB=megallm
  
  OPENROUTER_API_KEY=sk-or-v1-...  (for blog generation)
  OPENROUTER_MODEL=qwen/qwen3.6-plus-preview:free
  USE_OPENROUTER=true
  
  MEGALLM_API_KEY=sk-mega-...  (for WF1 insights)

Blog generation settings in config.py:
  BLOG_WORD_COUNT_MIN = 600
  BLOG_WORD_COUNT_MAX = 900
  BLOG_TEMPERATURE = 0.65

TROUBLESHOOTING
═════════════════

No insights found?
  └─ Ensure scraping + WF1 ran first
  └─ Check: db.content_insights.find().count()

MongoDB connection error?
  └─ Start MongoDB: mongod
  └─ Check MONGODB_URI in .env

API key error?
  └─ For blogs: Check OPENROUTER_API_KEY in .env
  └─ For WF1: Check MEGALLM_API_KEY in .env

Generation failed for account?
  └─ Check app.py logs for specific error
  └─ Try running step 3 alone first

Slow generation?
  └─ Normal: API calls take 10-30 seconds each
  └─ 75 blogs = ~20 minutes total
  └─ No optimization yet (could add batching)
