🚀 4-Step Startup

Step 1: Install Python Packages

cd blog_platform pip install -r requirements.txt

Step 2: Configure Environment

copy .env.example .env (Windows) cp .env.example .env (Mac/Linux)

Edit .env and add your API key:

MEGALLM_API_KEY=sk-mega-YOUR_KEY_HERE MONGODB_URI=mongodb://localhost:27017 MONGODB_DB=megallm_blog_platform

Step 3: Start MongoDB

Option A: If installed locally

mongod

Option B: Using Docker

docker run -d -p 27017:27017 mongo

Step 4: Start the Platform

python main.py

Wait for message: "Platform initialized successfully!"

Then visit: http://localhost:5000

✓ Verify Everything Works

Before starting, run the setup verification:

python setup.py

This will check:

  • ✓ All Python packages installed
  • ✓ MongoDB connection working
  • ✓ .env file configured properly
  • ✓ API key is valid and has credits

🖥️ Command-Line Tool (Optional)

Don't want to use the web dashboard? Use the CLI instead:

python cli.py accounts # List all 5 accounts python cli.py status account_1 # Show statistics python cli.py list account_1 # Show all blogs python cli.py generate account_1 # Generate 12 blogs now python cli.py show <blog_id> # Show full blog content python cli.py mark-posted <blog_id> # Mark as published

See python cli.py --help for all commands.

⚙️ Configuration Options

All settings in config.py:

  • Blog word count: 500-800 words (adjustable)
  • Generation schedule: Every 2 hours (configurable)
  • Blogs per 24h: 12 total (3 per topic)
  • LLM model: deepseek-ai/deepseek-v3.1 (can swap)
  • Accounts: 5 predefined (can add more)
  • Topics: 4 CTO-focused categories (customizable)

💾 Database Info

MongoDB collections (auto-created):

  • accounts - 5 accounts + metadata
  • blogs - Generated blog posts
  • generation_history - When blogs were generated

All data persists in MongoDB. No data lost on app restart.

🐛 Quick Troubleshooting

"MongoDB connection failed"

mongod # Start MongoDB mongo --eval "db.adminCommand('ping')" # Verify it's running

"API key error" or "402 Insufficient Credits"

Check your .env file has a valid key starting with sk-mega-

Test it: Run python setup.py and check "Testing API Key"

"No blogs generating"

  1. Click "🚀 Generate Blogs Now" manually
  2. Wait 2-3 minutes (API takes time)
  3. Check blog_platform.log for errors

"Cannot access http://localhost:5000"

  1. Verify Flask started (check console output)
  2. Port 5000 might be in use: Change in main.py
  3. Try health check: http://localhost:5000/api/health
