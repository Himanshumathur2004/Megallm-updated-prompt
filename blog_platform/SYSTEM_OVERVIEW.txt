API ENDPOINTS:
==============

Web:
  GET  /                           → Dashboard UI

Accounts:
  GET  /api/accounts               → List all 5 accounts
  GET  /api/accounts/<account_id>  → Get single account

Blogs:
  GET  /api/blogs?account_id=...   → List blogs (filter by status)
  GET  /api/blogs/<blog_id>        → Get single blog
  GET  /api/blogs/<blog_id>/copy   → Get copy-ready content
  POST /api/blogs/generate         → Generate new blogs
  PUT  /api/blogs/<blog_id>/mark-posted  → Mark as posted
  DELETE /api/blogs/<blog_id>      → Delete draft blog

Dashboard:
  GET  /api/dashboard/<account_id>       → Statistics summary
  GET  /api/generation-history/<account_id> → Generation events

Health:
  GET  /api/health                 → Status check


GENERATION FLOW:
================

Automatic Schedule (Every 2 Hours):
1. Scheduler triggers for each account
2. Loops through 4 topics
3. Generates 1 blog per topic (3 total, but scheduled across day)
4. Inserts into blogs collection with status="draft"
5. Logs event in generation_history
6. Updates account.last_generation

Manual Trigger:
1. User clicks "🚀 Generate Blogs Now"
2. Same process as above
3. Supports custom topic selection (API)

Per Blog Generation:
1. BlogGenerator.generate_blog() is called
2. Uses MegaLLM API with detailed system prompt
3. Returns title + body
4. Validates JSON response
5. Inserts to MongoDB with metadata
6. Returns blog_id

COMMAND-LINE INTERFACE:
=======================

Optional CLI tool (cli.py) for managing without web UI:

python cli.py accounts              # List all accounts
python cli.py status account_1      # Account statistics
python cli.py list account_1        # List blogs
python cli.py list account_1 --status=draft  # Filter by status
python cli.py generate account_1    # Generate 12 blogs
python cli.py generate account_1 --count=5   # 5 per topic
python cli.py show <blog_id>        # Show full content
python cli.py mark-posted <blog_id> # Mark as published
python cli.py delete <blog_id>      # Delete draft blog
python cli.py export account_1      # Export to JSON

PRODUCTION CONSIDERATIONS:
==========================

Before deploying to production:

1. Change SECRET_KEY in .env
2. Set DEBUG=False in .env
3. Use production MongoDB (MongoDB Atlas)
4. Set FLASK_ENV=production
5. Use process manager (Gunicorn, PM2)
6. Set up reverse proxy (Nginx)
7. Enable HTTPS/SSL
8. Set up monitoring & logging
9. Configure rate limiting
10. Add authentication if needed
