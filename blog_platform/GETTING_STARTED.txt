✅ GETTING STARTED CHECKLIST
==============================

Your blog generation platform is ready! Follow these steps to launch:


PHASE 1: SETUP (5 minutes)
==========================

□ 1. Open terminal and navigate to blog_platform:
     cd blog_platform

□ 2. Install Python dependencies:
     pip install -r requirements.txt
     
     Expected: "Successfully installed flask, pymongo, requests, apscheduler..." 

□ 3. Copy configuration file:
     copy .env.example .env     (Windows)
     cp .env.example .env       (Mac/Linux)

□ 4. Edit .env and add your API key:
     Open .env in text editor
     Find: MEGALLM_API_KEY=sk-mega-YOUR_KEY_HERE
     Replace with your actual key
     Save the file

□ 5. Verify MongoDB is installed:
     mongod --version
     (If not installed, download from mongodb.com)

□ 6. Start MongoDB in a separate terminal:
     mongod
     
     Expected: "waiting for connections on port 27017"


PHASE 2: VERIFY (2 minutes)
============================

□ 7. Run verification script:
     python setup.py
     
     Should show:
     ✓ Requirements - All satisfied
     ✓ MongoDB - Running on localhost:27017
     ✓ .env Configuration - All keys present
     ✓ API Key - Valid and has credits

□ 8. If any checks fail:
     • MongoDB not found → Start mongod
     • API key error → Check .env file
     • Packages missing → Run pip install again
     • Read error messages carefully


PHASE 3: LAUNCH (1 minute)
===========================

□ 9. Start the platform:
     python main.py
     
     Expected output:
     ================== BLOG GENERATION PLATFORM - STARTUP ===================
     Starting background scheduler...
     MegaLLM Model: deepseek-ai/deepseek-v3.1
     .......................... Ready to Start! ..........................
     Web UI: http://localhost:5000

□ 10. Leave this terminal running
      (Don't close it - platform runs in background)


PHASE 4: FIRST USE (5 minutes)
===============================

□ 11. Open web browser:
      http://localhost:5000
      
      You should see: Beautiful dashboard with 5 account buttons

□ 12. Click on an account:
      Example: Click "Account 1" button
      
      You should see: Statistics (0 blogs initially)

□ 13. Generate your first blogs:
      Click "🚀 Generate Blogs Now" button
      
      Wait 2-3 minutes (API takes time to generate)
      Then refresh or wait for auto-update (30 seconds)

□ 14. First blogs appear!
      You should see: 12 new blog cards in the list
      
      Each showing:
      • Title (green topic tag, blue status badge)
      • Preview text
      • Date created
      • 📋 Copy button
      • ✅ Mark Posted button

□ 15. Test copy functionality:
      Click "📋 Copy" on any blog
      
      Modal appears with:
      • Title section - "Copy Title" button
      • Body section - "Copy Body" button
      
      Click "Copy Title" → Check clipboard (Ctrl+V somewhere)
      Click "Copy Body" → Check clipboard

□ 16. Test mark posted:
      Click "✅ Mark Posted" button
      
      Blog status changes to "POSTED" (green badge)
      Posted count increases in statistics

TROUBLESHOOTING QUICK FIXES
============================

Problem: Page won't load (localhost:5000)
→ Check terminal running main.py is still active
→ Try http://localhost:5000/api/health
→ If flask not started, check for port 5000 conflicts

Problem: "No blogs appearing after click"
→ Give it 2-3 minutes (API is slow sometimes)
→ Refresh page after waiting
→ Check browser console for errors (F12)
→ Check blog_platform.log for error messages

Problem: "API Error 402"
→ Your API key likely has no credits
→ Check MEGALLM_API_KEY in .env
→ Should start with "sk-mega-"
→ Get a new key from MegaLLM if insufficient credits

Problem: "MongoDB connection refused"
→ MongoDB not running
→ Open new terminal and run: mongod
→ Wait for "waiting for connections on port 27017"

Problem: "ModuleNotFoundError: No module named 'flask'"
→ Dependencies not installed
→ Run: pip install -r requirements.txt
→ Make sure you're in blog_platform directory


CUSTOMIZATION OPTIONS (Later)
============================

Once running, you can edit config.py to:
□ Add more accounts (edit ACCOUNTS list)
□ Add more topics (edit TOPICS dict)
□ Change generation frequency (edit GENERATION_INTERVAL_MINUTES)
□ Change blog length (edit BLOG_WORD_COUNT_MIN/MAX)
□ Use different LLM model (edit MEGALLM_MODEL)

For each change, restart main.py
